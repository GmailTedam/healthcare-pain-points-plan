# BulletTrain — FHIR Simulator Retirement

**Repo:** `~/bt/BulletTrain`
**Depends on:** `~/health-agent-workspace/infra/fhir/` (workspace HAPI + Caddy, already shipped)
**Status:** Not started. Planned follow-up after workspace FHIR rebuild (2026-04-07).

## 1. Context
BulletTrain currently boots three hand-rolled FastAPI "simulators" that speak FHIR-ish JSON:

| Simulator | Path | Port | pack.json |
|---|---|---|---|
| `ehr_hims` | `services/ai_simulators/external_systems/ehr_hims/` | 9104 (runtime), 8814 (stale YAML) | `bt_ai_agent_test/connector_packs/ehr_hims/1.0.0/pack.json` |
| `client_registry` | `services/ai_simulators/external_systems/client_registry/` | 9102 (runtime), 8891 (stale YAML) | `bt_ai_agent_test/connector_packs/client_registry/1.0.0/pack.json` |
| `terminology_service` | `services/ai_simulators/external_systems/terminology_service/` | 9117 (runtime), 8895 (stale YAML) | `bt_ai_agent_test/connector_packs/terminology_service/1.0.0/pack.json` |

`client_registry` and `terminology_service` already carry `DEPRECATED.md`. `ehr_hims` is actively modified and load-bearing in Playwright E2E.

**Boot sites:**
- `scripts/start_test_services.py` (lines 79–80, 88, 114–115)
- `frontend/playwright.global-setup.ts` (lines 75–87, 150–152)

**Stale YAML:** `config/services.generated.yaml` is generated by `services/connector_registry/pack_runtime.py` from `bt_ai_agent_test/connector_packs/*/*/pack.json`. Its `base_url` values (8814/8891/8895) do not match actual runtime ports (9102/9104/9117) — the generator output is already drifted.

**Dashboard seed:** `services/api_gateway/integrations.py:40` declares `fhir_proxy` dsn `https://api.bullettrain.local/fhir` with a fake `status: "online"` literal and no health probe.

**Rule (workspace memory `feedback_fhir_real_services.md`):** never use BT simulators; default to real FHIR services. The workspace HAPI behind Caddy at `https://bullettrain.local/fhir` is now the canonical real target.

## 2. Goal
Retire the three FHIR simulators. Every BT call site and test that currently hits them must instead hit the workspace HAPI at `https://bullettrain.local/fhir` (or `https://api.bullettrain.local/fhir`), without loss of test coverage.

## 5. Implementation plan

### Phase A — Inventory and baseline
1. Capture a BT test baseline: `pytest tests/ -q`, `pnpm -C frontend test`, and a full Playwright E2E run. Save results under `out/bt-retirement-baseline/`. Every retired simulator endpoint must map to tests that currently pass.
2. `rg -n "ehr_hims|client_registry|terminology_service|9102|9104|9117|8814|8891|8895|fhir_proxy"` across BT. Build a call-site matrix: file, line, endpoint used, test it backs.
3. Diff every endpoint each simulator currently serves against the HAPI CapabilityStatement. Identify gaps — assertions the simulators satisfy that real HAPI won't without IG loading.

### Phase B — IG loading on workspace HAPI
4. Pull UK Core IG package (and any others the simulators fake) from simplifier.net: `POST /fhir` with `ImplementationGuide` Bundle against `https://bullettrain.local/fhir`.
5. Flip `HAPI_FHIR_VALIDATION_RESPONSES_ENABLED=true` in `infra/fhir/docker-compose.yml` once IGs are loaded; re-run REA suite to confirm nothing regresses.
6. If the simulators expose fixture data (e.g. seeded patients), seed equivalent Patient/CarePlan/etc. resources into HAPI via `POST /fhir/Patient` so BT assertions on known IDs still match.

### Phase C — Retarget BT tests
7. Add `BT_FHIR_BASE_URL=https://bullettrain.local/fhir` to BT's test env (`.env.test`, `frontend/.env.test`, CI secrets). Default fallback: `https://bullettrain.local/fhir`.
8. Replace every direct call to `127.0.0.1:9102/9104/9117` with a reference to `BT_FHIR_BASE_URL`.
9. Replace every import of the FastAPI simulator modules with nothing — the symbols go away.
10. Update `frontend/playwright.global-setup.ts` to stop booting the three simulators. Add a pre-flight probe of `https://bullettrain.local/fhir/metadata` that fails fast with a clear message if the workspace stack isn't up.
11. Update `scripts/start_test_services.py` similarly.
12. Run BT unit + integration + Playwright. Fix fallout until green. Expected friction: HAPI is stricter about cardinality / profile conformance than the FastAPI doubles. Fix is either tighter fixtures or stricter assertions — not relaxing HAPI.

### Phase D — Delete simulators
13. Rollback tag: `git tag bt-pre-fhir-sim-retirement`.
14. Delete `services/ai_simulators/external_systems/{ehr_hims,client_registry,terminology_service}/`.
15. Delete `bt_ai_agent_test/connector_packs/{ehr_hims,client_registry,terminology_service}/1.0.0/pack.json` (or retarget to reference the workspace HAPI — decide per-pack based on whether the pack_id is still a meaningful logical service).
16. Run the pack runtime to regenerate `config/services.generated.yaml`. Assert the three entries are gone.
17. Update `services/api_gateway/integrations.py`: replace `fhir_proxy` seed dsn, remove the hardcoded `status: "online"`, add a live health probe against `/fhir/metadata`. Dashboard UI must reflect real state.

### Phase E — Verification and docs
18. Full BT test suite green: unit + integration + Playwright E2E.
19. `services.generated.yaml` diff matches expectations; no FHIR entries.
20. `integrations.py` dashboard UI shows `fhir_proxy` with a real health timestamp updated by the probe.
21. Update BT `README.md` + `AGENTS.md` to point at the workspace FHIR stack as a prerequisite.
22. Update BT `CLAUDE.md` to reference `feedback_fhir_real_services.md` memory rule.

## 7. Rollback
Single `git revert bt-pre-fhir-sim-retirement..HEAD` restores the simulator tree and the boot sites. Workspace HAPI stack is unaffected by BT rollback.

## 8. Risks
- **HAPI strictness vs simulator leniency.** The simulators accept payloads real HAPI would reject. Loading IGs helps but cannot fully fix assertion drift. Budget a day for this.
- **Playwright E2E flakes.** Cold-starting HAPI + Postgres + Caddy takes ~30–60s. CI jobs need a warm cache or longer pre-start.
- **Fixture seed drift.** If the simulators return hardcoded patient IDs the BT tests assert on, those IDs must be seeded into HAPI deterministically (idempotent seed script, run pre-test).
- **Hosts file requirement.** Any machine running BT E2E must have `bullettrain.local` resolving to 127.0.0.1. Either run `infra/fhir/add-hosts.ps1` as admin, or apply the pytest resolver shim pattern from REA-Agent-mcp/tests/conftest.py, or both.

## 9. Non-goals
- Retiring non-FHIR simulators (REST, HL7v2, DICOM, X12, BATCH, GRPC) — out of scope. User rule is FHIR-only.
- Changing BT's connector manifest schema or pack format.
- Touching REA-Agent-mcp (already migrated).
